Trajectory of Taxis¶

Here we'll explore taxi trip data collected from Beijing in 2008. It's described in

https://www.microsoft.com/en-us/research/publication/t-drive-trajectory-data-sample/

First things first¶

It turns out that there are missing links to the data files from that website. For convenience, I've repacked them in the current directory (t-drive-sample-part1.zip, t-drive-sample-part2.zip). To run the notebook (taxis.ipynb), you'll need to unzip those files.

The missing urls were easy to infer. The files are labeled ranging from 01.zip to 014.zip starting at the url: https://www.microsoft.com/en-us/research/wp-content/uploads/2016/02/01.zip

Using all the available data gives a better view of the details. For example here's a view of Beijing via the taxi data. The code to reproduce this is below.

(1.B) Graph distance intervals.¶

Calculate GPS distances¶

Next we'd like to compute incremental distances between GPS points. This dataset is large enough that ordinary python code might run too slow. There are a number of algorithms and tools for computing the distance between two GPS points. In fact, I tested several tools from python libraries like geopy (vincenty) and geographiclib (Geodesic.WGS84.Inverse). I found them to be far too slow to use (they'd take at least a day to finish!). Instead, I chose a simple numerical formula written in vectorized python (numpy) for speed.

In [9]:
# Use numpy functions for the distance formula

def gps_dist(a, b, c, d):    
    '''Compute the distance (in meters) between two gps locations. Input is assumed to be a = longitude, b = latitude, etc.'''
    r = 0.0174533  # 1 degree in radians
    return 2 * 6371000 * np.arcsin( np.sqrt( # https://en.wikipedia.org/wiki/Haversine_formula
        np.sin(r*(d - b)/2.0)**2 + np.cos(r*b) * np.cos(r*d) * np.sin(r*(c - a)/2.0)**2))
